# Z-MAX 类脑计算方案

> 基于 SmolVLA 脑科学映射，Z-MAX 三层解耦架构的产品化落地

---

## 方案1: 🧠 类脑VLA推理引擎

### 脑科学依据

```
人脑双通路:
  腹侧通路(V1→IT): "这是什么" → 识别任务场景
  背侧通路(PPC→M1): "往哪动" → 生成动作序列

SmolVLA映射:
  VLM(350M冻结) = 腹侧通路 → 场景理解
  Expert(98M训练) = 背侧通路 → 动作决策
  Cross-Attention   = 两条通路间的信息交换
```

### Z-MAX产品化

| 特性 | 实现 |
|------|------|
| 场景理解 | SmolVLM2-500M 预训练VLM (冻结) |
| 动作决策 | 98M Expert, 交叉注意力读VLM |
| 快速微调 | 仅训练Expert 100M参数 (全量450M的22%) |
| 推理速度 | ~270ms/帧 (RTX4060) |
| 新场景适配 | 微调Expert即可, VLM复用 |

## 方案2: 🎯 三层解耦 + 快速领域迁移

### 脑科学依据

```
大脑三层:
  感觉皮层(理解) → 联合皮层(规划) → 运动皮层(执行)
  
  换一个任务: 感觉皮层不变(视觉一样)
              联合皮层微调(新任务规则)
              运动皮层微调(新动作模式)
```

### Z-MAX产品化

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: 感知层 (Sys-2 数据系统)                     │
│  • 数据集管理 (PushT / MetaWorld / Orin真机)        │
│  • 图像预处理 (480×480 RGB → VLM输入)               │
│  • 数据增强 + 缓存管理                               │
├─────────────────────────────────────────────────────┤
│ Layer 2: 认知层 (Sys-11 训练系统)                    │
│  • SmolVLA VLM冻结 + Expert微调                     │
│  • 30+参数可视化控制台                               │
│  • 训练记录 + 评估模块                               │
├─────────────────────────────────────────────────────┤
│ Layer 3: 执行层 (Sys-12 推理系统)                    │
│  • gRPC推理服务 (Server/Client分离)                 │
│  • 旁路验证 (本地Server + 远端Client)               │
│  • ROS2 Gateway桥接                                  │
└─────────────────────────────────────────────────────┘
```

## 方案3: 🌐 旁路推理验证

### 设计理念

```
人脑的感觉-运动分离:
  感觉输入 → 大脑处理 → 运动输出
  
  旁路验证:
  观察数据 → Server推理 → 动作指令
       (独立通道, 不与主控冲突)
```

### Z-MAX架构

```
Orin真机 ──SSH──→ Mac Gateway ──gRPC──→ 本地Server(SmolVLA)
  ↑                                         │
  │  /real_joint_states                      │ 动作指令
  │  /gripper_pos                            ↓
  └──────────────────── 执行 ────────────────┘
```

| 组件 | 部署位置 | 职责 |
|------|---------|------|
| Server | WSL(RTX4060) | 加载SmolVLA, 推理 |
| Gateway | Mac M1 | SSH中转, HTTP API |
| Client | 本地/Orin | 发送观测, 接收动作 |
| ROI旁路 | 独立进程 | 不干扰主控PLC |

## 方案4: ⚡ Expert快速微调 (100M方案)

### 脑科学依据

```
大脑学习:
  不重学视觉(太贵) → 只调整运动策略(快)
  
  基底节-小脑: 新动作通过多巴胺快速调整突触权重
              不需要重建整个视觉通路
```

### Z-MAX落地

```
SmolVLA微调配置:
  freeze_vision_encoder: true     ← VLM冻结(识物能力保留)
  train_expert_only: true         ← 只练Expert(动作策略)
  
  训练参数: 100M / 450M (22%)
  训练速度: 4× faster
  显存需求: ~2GB (RTX4060 8GB够用)
  
  适用场景:
  • PushT → 推块任务
  • MetaWorld → 多任务操作
  • 光模块插拔 → 精细操作
```

## 方案5: 🔮 下一代架构方向 (从脑科学借鉴)

| 脑科学发现 | 当前SmolVLA缺失 | Z-MAX下一代方向 |
|-----------|----------------|----------------|
| 处处反馈(V1↔IT↔PPC) | 纯前馈 | 加循环反馈层 |
| 预测编码(持续预测+误差修正) | 一次Flow Matching | 多步预测+校正 |
| 触觉-视觉融合 | 只有视觉+语言 | 加触觉编码器(Z-MAX已预留) |
| 多巴胺在线学习 | 离线训练 | 部署后在线微调 |
| 稀疏激活(~1%神经元) | 全激活 | 混合专家(MoE) |
