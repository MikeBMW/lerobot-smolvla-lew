## 版本号规范

```
完整版本号 = {LeRobot版本}-zmax.{Z-MAX版本}

示例:
  0.5.2-zmax.1.0.0

含义:
  - 0.5.2  → 对应的 LeRobot 上游基线版本 (pyproject.toml)
  - zmax.  → 固定标识符
  - 1.0.0  → Z-MAX 自定义版本 (语义化版本号)
```

### Z-MAX 版本号规则 v{X}.{Y}.{Z}

| 段 | 含义 | 变更触发 |
|----|------|---------|
| **X** (主版本) | 产品架构变更 | 硬件平台升级、技术路线重大调整 |
| **Y** (次版本) | 功能增加 | 新增Phase、新增场景、ROI模型更新 |
| **Z** (修订版) | 错误修正 | 指标修正、文档勘误 |

### 文档版本一致性要求

> **三层文档（L1/L2/L3）必须保持版本号一致。** 每次版本发布前，由产品经理确认以下"一致性锚点"在三份文档中对齐：

| 锚点 | 当前值 (v1.0.0) |
|------|-----------------|
| 产品名称 | Z-MAX 多模态动作专家 |
| 硬件平台 | Z700 轮式双臂 |
| 插拔精度 | ±0.02mm |
| 成功率 | >99% |
| 力控带宽 | >10kHz |
| 双臂协同 | 左取料 + 右插拔 |
| 技术路线 | Phase 0(L2) → Phase 1(L3) → Phase 2(L3+) → Phase 3(L4) → Phase 4(L4+) |
| 技术架构 | VLM规划 + VLA执行 + HIL强化学习 |
| ROI回收期 | 14~22月 |

## 版本发布流程

1. **产品经理更新**：同步修改 L1/L2/L3 三份文档，确认一致性锚点对齐
2. **GUI版本号**：更新 `tools/gui/studio.py` 和 `tools/gui/version_sync.py` 中的 `zmax_ver` 值
3. **归档旧版本**：将上一版本文件移入 `docs/archive/`
4. **提交发布**：
   ```bash
   git add docs/ tools/gui/studio.py tools/gui/version_sync.py
   git commit -m "release: Z-MAX v{X}.{Y}.{Z} — {说明}"
   git push origin main
   ```
