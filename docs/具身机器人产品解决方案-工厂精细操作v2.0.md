## 执行摘要

### 战略背景

2026年2月，工信部正式发布《人形机器人与具身智能标准体系（2026版）》，首次以国家级顶层设计覆盖具身智能全产业链与全生命周期标准，确立了中国在具身智能产业化进程中的制度框架地位。同年，具身智能写入政府工作报告，与6G、量子技术并列为国家重点未来产业。截至2025年年中，全年投资事件已超91起、总规模逾110亿元人民币（数据来源：行业投研报告综合统计，详见附录D），产业正式从"实验室Demo"阶段迈入"垂类规模落地大年"。

高速光模块产业是AI算力基础设施的核心硬件载体，受800G/1\.6T光模块需求驱动，国内外量产扩产节奏持续加快。然而，高速光模块的测试与装配环节至今仍高度依赖人工，人工插拔力度不可控导致良率受损、换型耗时数小时无法柔性生产、狭小密集型测试仓难以作业——三大痛点叠加制约产业规模化提速。

本规范面向上述痛点，提出以Z\-MAX多模态动作专家为核心的具身智能机器人产品方案，目标实现 **L4级全自主精细操作**，产线连续作业成功率≥99\.2%，单机替代2\\\~3名熟练操作工，兼容100G/400G/800G/1\.6T/硅光全品类混线生产。

## 一、产品定义与市场战略

### 1\.1 产品定位

**精细操作具身机器人**，垂直落地高速光模块量产工厂，解决行业四大核心痛点：

|痛点|描述|当前后果|
|---|---|---|
|人工损伤不可控|人工插拔力度波动大，金手指划伤、光纤端面污染频发|良率受损，返工成本高|
|换型刚性成本高|传统专机仅支持单一型号，换型调试耗时4小时以上|柔性生产无法实现|
|环境适应性差|老化仓、测试仓空间密集狭小，人工及现有设备难以作业|自动化渗透率极低|
|规模化复制难|进口设备单价35万元起，落地门槛高|中小厂无法承受|

**核心产品形态：Z\-MAX 智蜂\-多模态动作专家**

命名逻辑解析：

- **智蜂（Z\-Bee）**：精密、高效、协同的工业级品牌IP

- **Z\-MAX** 四维含义：

    - **Z**（潜在特征空间）—— L4级全局感知与认知底座

    - **M**（多模态）—— L4级感官融合与自适应泛化

    - **A**（Action动作）—— L4级精细力控与动态执行

    - **X**（Expert专家）—— L4级自主决策与全自主闭环

聚焦四大核心工序：**光模块固件加载插拔、老化仓上下料、模块测试插拔、晶圆摆盘**。

对标行为智能（Behavior Intelligence）FAIR展会真机无干预全自主插拔落地能力，确立"首版可展示、可复测、可复制"的落地原则。

#### 轮式构型在工业精细操作场景的通用卖点

Z-MAX选择轮式构型而非双足构型，并非技术妥协，而是基于工厂场景的最优工程决策。以下是轮式构型相对于双足构型在光模块精细操作场景中的核心优势：

|对比维度|轮式构型（Z-MAX选择）|双足构型|工业场景结论|
|---|---|---|---|
|**运动稳定性**|底盘全接触地面，无平衡控制开销|需实时平衡校正，重心波动大|轮式在插拔操作时末端抖动更小|
|**负载能力**|载荷直接传导至地面，可承载20kg+双臂操作|载荷受限于平衡能力，通常<10kg|轮式可搭载重载双臂+传感器套件|
|**部署速度**|落地即用，无需地面适应性调试|需评估地面摩擦、平整度、台阶|轮式从开箱到投产时间缩短70%|
|**续航效率**|滚动摩擦系数低（~0.01），同电池续航长3x|步行能耗高（维持平衡本身耗能）|轮式8h连续作业无需充电|
|**成本结构**|成熟AGV底盘，BOM成本可控|高精度关节电机+IMU，BOM高40%+|轮式单机成本下降，适合RaaS模式|
|**安全合规**|低速滚动+碰撞检测，人机混线合规容易|跌倒风险高，需安全围栏隔离|轮式在无尘车间无需围栏部署|

> **注意**：轮式构型的局限性在于无法跨越台阶和复杂地形，但光模块工厂车间地面平整度标准（±2mm/m）完全满足轮式需求。Z-MAX的ODD边界明确排除了非平整地面场景，因此轮式局限性不影响产品适用性。

## 三、系统总体方案设计

### 3\.1 总体设计目标

将系统功能设计锚定 **L4级**——产品不再是"执行动作的机械臂"，而是具备"感知\-决策\-执行"闭环的 **具身智能体**。

L4级与Z\-MAX四维度的对应关系：

|Z\-MAX维度|L4级核心挑战|技术支撑方式|
|---|---|---|
|Z（潜在空间）|复杂环境深度理解|多源传感器→高维特征空间映射|
|M（多模态）|全品类无缝混线|视觉\+力觉\+本体觉融合，零/少样本泛化|
|A（Action）|亚毫米精细力控|实时生成毫米级操作轨迹\+力控补偿|

### 5\.2 SmolVLA：轻量级视觉语言动作模型（Sys\-11实现）

**定位：** 面向消费级GPU（RTX 4060 / Jetson Orin）优化的轻量化VLA模型，作为Sys\-11快系统核心引擎。

**技术选型理由：**

|对比维度|VLA\-JEPA（原方案）|SmolVLA（当前选型）|选择理由|
|---|---|---|---|
|视觉语言主干|Qwen3\-VL\-2B（20亿参数）|SmolVLM2\-500M（5亿参数）|**参数量减少75%**，端侧部署友好|
|推理显存需求|\~12GB（RTX 4070\+）|\~8GB（RTX 4060可运行）|**适配现有硬件**|
|世界模型依赖|V\-JEPA2（必需）|可选关闭（LeWorldModel）|**降低训练复杂度**|
|推理延迟目标|\<10ms|\<10ms|满足实时控制要求|

**架构设计：**

```Plaintext
SmolVLA 架构（Sys-11 快系统）

输入: 图像 + 语言指令
         │
         ▼
┌─────────────────────────┐
│  SmolVLM2-500M          │  ← 轻量视觉语言主干
│  + Expert Model          │     (SigLIP视觉编码器 + 专家层)
│  (冻结VLM, 仅训练Expert) │
└────────────┬────────────┘
             │ multimodal_embeds [B, seq_len, hidden_dim]
             ▼
┌─────────────────────────┐
│  DiT-B Action Head      │  ← flow-matching扩散模型
│  (2层Transformer)       │     预测动作序列
└────────────┬────────────┘
             │
             ▼
      动作输出 [B, 7, action_dim]
```

**关键改造点（基于vla\_jepa重构）：**

1. **视觉语言主干替换**：

    - 移除: `Qwen3VLForConditionalGeneration` \(2B参数\)

    - 替换: `SmolVLMWithExpertModel` \(500M参数 \+ Expert层\)

    - 优势: 冻结VLM主干，仅训练Expert层，显存占用降低50%

2. **Token工程简化**：

    - 移除: 自定义action token扩展（`<|action_0|>`, `<|embodied_action|>`）

    - 替换: SmolVLM原生多模态处理，无需手动扩展tokenizer

    - 优势: 减少tokenizer hack，训练更稳定

3. **世界模型解耦**：

    - 原方案: V\-JEPA2强制启用（训练时预测视频帧）

    - 当前方案: LeWorldModel可选关闭（`enable_lew_world_model=False`）

    - 优势: L3阶段可独立验证SmolVLA性能，降低训练复杂度

4. **动作头复用**：

    - 完全保留: DiT\-B flow\-matching架构

    - 推理: 4步欧拉去噪生成动作序列

    - 优势: 算法成熟，与vla\_jepa共享代码

**工程验证状态：**

- ✅ RT 4060 \(8GB显存\) 训练验证通过

- ✅ LeRobot框架集成（`SmolVLALewPolicy`）

- 🔄 Jetson Orin Nano 部署进行中（量化优化）

### 5\.3 LeWorldModel \(LeWM\)：轻量世界模型（Sys\-12实现）

**定位：** 面向因果理解与3D空间推理的轻量世界模型，作为Sys\-12慢系统核心引擎，为SmolVLA提供目标位姿引导。

**技术选型理由：**

|特性|数值|工程意义|
|---|---|---|
|参数量|**15M（1500万）**|端侧部署无压力，可运行于Jetson Orin|
|训练时间|**单GPU数小时**|快速迭代验证，适配光模块垂类场景|
|推理速度|比通用基础模型快 **48倍**|满足10\-20Hz认知推理频率要求|
|损失函数|预测损失 \+ 高斯正则化|超参数从6个缩减至**1个**|
|物理直觉|编码真实物理结构|可检测违背物理的异常事件（"惊奇度"）|

**在Z\-MAX系统中的角色：**

```Plaintext
Sys-12 (LeWorldModel) 工作流

输入: 当前观测图像 I_t + 语言指令
         │
         ▼
┌─────────────────────────┐
│  LeWorldModel Encoder   │  ← 15M参数，轻量级
│  (JEPA架构)             │     潜空间预测，非像素重建
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│  3D空间推理             │  → 输出: 目标位姿引导

└─────────────────────────┘
             │
             ▼

```

- **L3阶段**: LeWorldModel可选关闭，SmolVLA独立工作（结构化场景）

- **L4阶段**: LeWorldModel启用，为SmolVLA提供非标场景的位姿引导

- **频率解耦**: LeWorldModel 10\-20Hz → SmolVLA 100Hz\+，异步协作

**工程验证状态：**

### 5\.4 力控搜索算法：李萨如轨迹\+柔性力控融合

**应用场景：** 光模块插入EVB/老化架时，力度动态自适应，硬壳模块5\\\~15N，杜绝过压损坏金手指。

**算法原理：**

```Plaintext
力控搜索算法流程

  ┌──────────┐     ┌───────────┐     ┌──────────┐
  │ 李萨如    │     │  柔性力控  │     │  装配锁定 │
  │ 轨迹规划  │────→│  实时阻抗  │────→│  确认到位 │
  │ (空间探索)│     │ (接触顺应) │     │  (力特征) │
  └──────────┘     └───────────┘     └──────────┘
                                         
  末端沿平滑曲线       接触力实时调节       到位确认
  进行空间搜索         产生微小偏移         力矩突变
                      平滑滑入目标         锁定释放
```

**技术优势：** 在确保精密零件零损伤的前提下，大幅提升复杂装配场景的搜索对准效率与良品率。

## 六、分级功能演进策略

> **标准依据**：本章分级定义严格对齐《智算中枢·智能化技术标准》（Q/ZFCY001.1-2026）第3章"系统功能分级"，并结合 Z-MAX 多模态动作专家在光模块精细操作场景中的工程实现进行落地展开。

### 6.1 智能化分级总览

针对光模块工厂中"光模块插拔"亚毫米级精密操作任务，智能化分级（L0~L5）共六个层级，其中 L0 为纯人工基线，L1~L5 为设备逐步演进路径。

#### 6.1.1 L1~L5 跨越定义（精简表述）

|等级|精简定义|关键跨越逻辑|
|---|---|---|
|L1|纯人工插拔，无自动化设备，全工序人工完成插装|人工→设备：设备开始介入执行|
|L2|人工辅助多型号，全靠人工做标定，无需值守等人管|设备→匹配：从单型号固化到人工辅助多型号切换|
|L3|自动匹配多型号，异常预警等人管|人工→自动匹配：系统自动识别并切换配置，释放人工标定时间|
|L4|划定产线随便混，出小问题自己搞|被动→自主处置：从"搞不定就停机等人"到"随机混产免匹配、异常自重试/报工单"|
|L5|全场景通用自主插拔，不限产线、品类、环境，跨车间跨工序自由调度，完全脱离固定产线ODD约束|定点→全域：从划定产线到跨车间跨工序，结合RoboOS调度策略|

#### 6.1.2 分级功能全景图

```Plaintext
L1 纯人工（基准对标）
    │  100%人工，无任何设备辅助
    ▼
L2 基础驱动底座+安全层+云端调度（BDP - Base Drive Platform）
    │  激活: System 0 + System 2
    │  模式: 驱动底座搭建 + 安全三级防护激活 + 云端任务下发
    │  特征: 单型号固定程序 + 人工辅助换型标定 + 可接收云端任务
    ▼
L3 条件自主插拔（CAP - Conditional Auto-Plug）
    │  激活: System 0 + System 1(基础) + System 2
    │  模式: VTLA（视触觉语言动作）模型自动匹配多型号 + ODD内自主执行
    │  特征: 自动识别型号切换参数 + 同系列自主微调
    ▼

    │  激活: System 0 + Sys-11(实时) + Sys-12(因果) + System 2
```

### 6.2 分级功能详细定义（L1~L5）

#### 6.2.1 L1级：纯人工插拔（Manual-only Baseline）

**定义**：完全依赖人工完成光模块插拔全工序，无任何自动化设备辅助。本层级仅作为**行业现状基准**，是 Z-MAX 产品的替代对象。

|维度|定义|
|---|---|
|功能范围|人工识别型号→手动摆放→人工插拔→人工目检→人工分类摆放，全工序100%依赖操作员|
|匹配方式|纯人工，无设备参与|
|异常处理|人工自行判断并处理|
|产线适用|任意产线、任意型号（完全依赖人工经验）|
|边界责任|良率、效率、损失全由人工承担|
|Z-MAX定位|**非产品层级，仅作为替代前的现状基准**，用于ROI计算中的"现状对照"和验收基线|

#### 6.2.2 L2级：人工辅助插拔（MAP - Manual Assist Plug）

**定义**：设备具备基础自适应能力，支持通过人工辅助匹配多种型号的光模块（如调用预设参数、人工微调等）；设备仅执行固定轨迹辅助插拔动作，操作员无需全程定点盯守，可跨工位巡检，但在设备发出异常告警时需及时响应并进行人工干预。

|功能清单|① 点位与工艺标定（人工示教+软件输入）<br>② 定点循环插拔（按标定轨迹执行，无自主上料）<br>③ 超限停机保护（超力/超程立即锁止，无微调）<br>④ 基础告警（本地声光报警，无远程推送）|仅System 0激活：HAL驱动+EtherCAT总线+三级安全防护|

**L2核心目的**：搭建可靠的硬件驱动底座与安全防护体系，**同时**采集高质量遥操作数据为L3级SmolVLA模型训练提供语料。数据采集是L2的**伴生副产出**，而非核心交付物。

#### 6.2.3 L3级：条件自主插拔（CAP - Conditional Auto-Plug）

**定义**：设备具备跨线自动配置与局部自适应能力，支持自动识别并匹配多种型号的光模块（一键切换程序参数）；在限定工况（ODD）内自主完成插拔与轻微异常处理，操作员无需定点盯守，仅在设备越界或严重异常告警时按规定窗口期完成人工接管。

### 6.4 演进阶段详细说明

**阶段一：L2基线（驱动底座与安全层搭建期）**

L2阶段的核心目的是**搭建可靠的硬件驱动底座、三级安全防护体系和云端任务调度能力**，确保机器人本体在任何极端情况下仍具备绝对安全的运动能力，同时可接收云端下发的基础任务。具体交付物包括：
1. **System 0驱动底座**：全向AGV轮式底盘 + 双7轴协作臂的HAL驱动层、EtherCAT实时总线、运动学正逆解全部调通
2. **三级安全层激活**：硬件急停（双冗余）+ 力控安全（≤50N全域停机）+ 软件降级保护全部验证通过
3. **System 2云端调度**：与云端智能体建立通信链路，可接收任务下发、状态上报、远程监控
4. **L3升级数据储备**：在驱动底座稳定运行的前提下，通过操作员高质量遥操作**同步采集"视觉-动作"对齐数据**，为后续L3级VTLA模型训练建立光模块垂类语料库

数据采集是驱动底座验证过程中的**伴生副产出**，而非L2的核心交付物。L2的核心交付物是**一台安全可靠、运动能力完备、可接受云端调度的硬件底座**。

**阶段二：L3突破（VTLA基础能力验证期）**

实现System 1(基础)能力：训练并部署VTLA（视触觉语言动作）模型，使机器人具备视觉识别、触觉感知、自然语言理解和动作执行的端到端能力。VTLA模型是Z-MAX智能能力的**基础版本**，为L4的实时化和因果推理能力奠定基础。此阶段重点：

1. 验证VTLA模型端侧推理能力（目标<50ms，为L4实时化预留优化空间）

2. 验证插拔控制精度（±0.02mm）

3. 验证跨SKU少样本泛化能力

4. 验证视触觉融合感知能力（视觉+力觉协同决策）

**L2→L3跨越验证**：确认"从人工辅助匹配多型号"到"VTLA模型自动匹配多型号"的跨越——操作员只需一键启动，VTLA模型自动识别型号并切换插拔工艺。

**阶段三：L4全链路（实时化改造 + 因果推理增强期）**

在L3的VTLA基础能力上进行两大升级：
1. **实时化改造（Sys-11）**：对L3的VTLA模型进行性能优化，将推理延迟从<50ms优化到<10ms，满足实时控制需求
2. **因果推理增强（Sys-12）**：引入LeWorldModel世界模型，提供因果推理能力（理解"为什么卡住"而非仅"检测到卡住"），实现故障自主诊断与处置

全链路打通：云端/系统2下发宏观任务→Sys-12(因果)理解场景并输出位姿引导→Sys-11(实时)结合实时多模态感知执行精细力控动作→状态反馈→异常自主处理。

**L3→L4跨越验证**：确认"从搞不定就停机等人"到"随机混产免匹配、搞不定自己重试/报工单"的跨越——划定产线内多型号随机混流，设备全程自主应对突发状况。

**阶段四：L5远期（全域自治运营期）** [远期迭代目标]

在L4的实时化+因果推理基础上，进一步实现跨场景零样本泛化与多机器人协同调度。结合RoboOS调度策略，实现多台Z-MAX跨车间跨工序自由调度，完全脱离固定产线ODD约束。此阶段是VTLA能力的全域泛化与分布式协同的终极形态验证。

### 8\.2 smolvla\-lew — 智蜂创元集成工程

**来源：** 智蜂创元自研（基于LeRobot框架，参考starVLA项目）

**定位：** Z\-MAX具身智能系统的核心软件集成层，打通SmolVLA \(Sys\-11\) \+ LeWorldModel \(Sys\-12\) \+ 硬件驱动，实现端侧部署。

**工程架构：**

```Plaintext
smolvla-lew 项目结构

lerobot/policies/smolvla_lew/
├── configuration_smolvla_lew.py   # SmolVLALewConfig (配置类)
├── modeling_smolvla_lew.py        # SmolVLALewModel + SmolVLALewPolicy (模型+策略)
├── action_head.py                 # SmolVLALewActionHead (DiT-B动作头)
├── processor_smolvla_lew.py       # 数据预处理/后处理流水线
└── __init__.py                    # 模块导出
```

**关键改造点（相对vla\_jepa）：**

|模块|vla\_jepa（原方案）|smolvla\_lew（当前方案）|改造理由|
|---|---|---|---|
|视觉语言主干|Qwen3\-VL\-2B|SmolVLM2\-500M \+ Expert|参数量\-75%，适配RTX 4060|
|世界模型|V\-JEPA2（强制）|LeWorldModel（可选）|降低训练复杂度|
|Token工程|自定义action token|SmolVLM原生处理|减少tokenizer hack|
|动作头|DiT\-B \(复用\)|DiT\-B \(复用\)|flow\-matching架构成熟|
|推理频率|\<10ms目标|\<10ms目标|满足实时控制要求|

**数据流设计：**

```Plaintext
LeRobot训练/推理流程

训练阶段 (forward):
  LeRobot batch (dict[Tensor])
        │
        ▼ _prepare_model_inputs()
  List[dict] native格式:
    - "image": List[PIL.Image]  多视角图像
    - "lang": str               任务指令
    - "action": [T, action_dim] 动作标签
        │
        ▼ SmolVLALewModel.forward()
  1. SmolVLM编码 → multimodal_embeds [B, seq_len, hidden_dim]
  2. DiT-B flow-matching → 预测动作速度场 → action_loss (MSE)
  3. (可选) LeWorldModel → 预测未来帧 → lew_loss (L1)
        │
        ▼
  total_loss = action_loss + 0.1 × lew_loss

推理阶段 (select_action):
  图像 + 指令
        ▼ SmolVLM编码
  multimodal_embeds
        ▼ DiT-B (4步欧拉去噪)
  动作序列 [B, 7, action_dim]
        ▼ Action Queue缓存
  逐步执行动作
```

### 8\.2 eb\_jepa — 联合嵌入预测架构开源库

**来源：** Meta FAIR（含Yann LeCun），Apache 2\.0许可

**仓库：** github\.com/facebookresearch/eb\_jepa（720\+ stars）

**论文：** arXiv:2602\.03604（ICLR World Models Workshop）

|示例任务|性能|意义|
|---|---|---|
|Image JEPA（CIFAR\-10自监督）|91%探测精度|从无标签数据学习视觉表征|
|Video JEPA（Moving MNIST）|稳定收敛|时序视频表征预测|

**与Z\-MAX集成方式：** 触觉信号通过Tactile VAE（触觉变分自编码器）模块编码并注入JEPA潜在空间，与视觉特征在潜空间对齐融合，无需修改V\-JEPA核心架构，以插件形式扩展多模态感知维度。

### 8\.3 MetaWorld — 多任务基准平台

**来源：** CoRL 2019（arXiv:1910\.10897），持续更新至2026年

50个机器人操作任务，涵盖抓取、按压、开门、抽屉、工具使用等，专门验证多任务学习与元强化学习泛化能力。其中涉及的动作组合（如"到达位置→转动→锁定"）与光模块插拔工序直接对应，可作为Z\-MAX模型训练前的仿真验证基准。

**最新进展：** 2026年5月，BISON系统（arXiv:2605\.15975）在扩展MetaWorld基准上验证了长程规划能力，证明该基准在2026年仍具活跃研究价值。
