## 二、变更流程规范

```
1. 提出变更 → GitHub Issue
2. 分支开发(feature/xxx) → 本地验证
3. PR提交 → 代码评审
4. 合并main → 打tag → 发布Release
5. 三工程同步更新(pull origin main)
```

## 三、ComfyUI 升级方向 (web负责)

### 3.1 保留功能(不可破坏)
- 🔗连接/▶执行/⏹停止 三按钮
- 节点拖拽·连线·缩放
- 3D张量可视化
- 日志面板

### 3.2 新增功能
- 顶部导航: [工作流] [监控面板] [版本历史] [设置]
- 工作流模板: 预设 dds_cycle.json 一键加载
- 状态栏: 版本号·更新时间·在线设备数·当前循环
- 多角色视图切换: 开发者/管理者/质检
- 暗色主题统一·响应式布局

### 3.3 UI标准
- 按钮间距≥8px·hover状态·loading动画
- 状态颜色: idle=灰色 running=绿色闪烁 error=红色 completed=蓝色
- 字体: 系统默认·等宽代码用Consolas
- 所有交互必须有反馈(不能点击无反应)
